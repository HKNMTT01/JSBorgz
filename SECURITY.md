# JETAMA PeopleOS Security Setup

## Why Supabase showed `UNRESTRICTED`

Supabase displays `UNRESTRICTED` when Row Level Security (RLS) is disabled on a table. The tables were created correctly by Django migrations, but Supabase warns that its Data API roles could potentially access them if privileges are available.

This system uses a server-rendered Django architecture:

```text
Employee browser → Django server → Supabase PostgreSQL
```

The browser must never receive `DATABASE_URL` or `SUPABASE_SERVICE_ROLE_KEY`.

## Secure all existing tables

After setting a real Supabase `DATABASE_URL` and running migrations:

```powershell
python manage.py migrate
python manage.py secure_supabase
```

Verify later with:

```powershell
python manage.py secure_supabase --check
```

The command:

- enables RLS on every table in the `public` schema;
- revokes table access from Supabase `anon` and `authenticated` roles;
- revokes sequence and function access from those API roles;
- leaves Django's direct PostgreSQL connection working.

Alternatively, paste `supabase_security.sql` into Supabase **SQL Editor** and run it once.

## Important architecture rule

Do not call HR tables directly from browser JavaScript with a Supabase anon key. All employee, leave, claim, attendance and audit operations must go through Django views with Django authentication and role checks.

## Storage

Create a bucket named `hr-private` and keep **Public bucket OFF**. Files are opened through short-lived signed URLs generated by Django using the server-only service-role key.

## Secrets

Never commit these values:

- `SECRET_KEY`
- `DATABASE_URL`
- `SUPABASE_SERVICE_ROLE_KEY`

The included `.gitignore` excludes `.env`.

## Production checklist

- Set `DEBUG=0`.
- Use HTTPS-only cookies.
- Set exact `ALLOWED_HOSTS` and `CSRF_TRUSTED_ORIGINS`.
- Rotate any key that was exposed in screenshots, chat, GitHub or frontend code.
- Use individual staff accounts; never share the administrator password.
- Enable Supabase backups/PITR according to company policy.
- Review audit logs and role assignments periodically.
